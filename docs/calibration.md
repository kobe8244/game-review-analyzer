# 情感分类校准实验记录

> 结论:通过人工标注校准与两轮 prompt 迭代,情感四分类准确率从 **90%(v1)提升至 92%(v3)**;
> 中间版本 v2(88%)的下降暴露了"规则矫枉过正"问题,由此细化出更精确的标注口径。

## 1. 实验设计

| 项目 | 设置 |
|------|------|
| 黄金数据 | 50 条人工标注(标注者:8 年游戏行业从业经验) |
| 抽样方式 | 从英文/繁中评论池(419 条)分层随机抽取:en 36 + zh-TW 14,`random_state=42` 可复现 |
| 盲标控制 | 标注样本中**不含 LLM 判断结果**,人工标注不受模型输出影响 |
| 任务 | 情感四分类:positive / negative / neutral / mixed |
| 模型 | deepseek-chat,temperature=0,JSON 输出 |
| 语言范围 | 仅 en / zh-TW(标注者可直接阅读的语言;es/pt/fr 未纳入校准,见局限) |

## 2. 结果

| 版本 | 准确率 | 说明 |
|------|--------|------|
| v1(zero-shot) | **90%**(45/50) | 无示例,仅任务描述。讽刺识别已表现良好 |
| v2(初版规则+few-shot) | **88%**(44/50) | 修复 4 条旧错,但规则过粗**新引入 5 条错**(见 §4) |
| v3(细化边界,最终版) | **92%**(46/50) | 相对 v1 修复 2 条、新引入 1 条 |

## 3. 标注口径(经人工校准确定)

1. **星级仅供参考,以文字实际表达为准**;讽刺、反话按真实含义判断
   (例:繁中区 1 星评论"你蠻幽默的" → negative,v1 即已正确识别)。
2. **mixed = 明确好评 + 抱怨现存的具体问题**(操作、强制账号、bug、延迟等)。
   "希望新增某功能/内容"属于**许愿**,不是问题抱怨:整体好评 + 许愿 → positive。
3. **通篇以抱怨为主**、正面只是顺带一提 → negative,不是 mixed。
4. **neutral = 有实质内容但通篇平铺直叙**(纯建议、与前作对比),无情绪词。
   "good"、"不错"等明确褒义短评是 positive,不算温吞。
5. **文字信息量过少无法判断时按星级折算**(纯表情、"okay"式短语):
   4~5 星 → positive,3 星 → neutral,1~2 星 → negative。

其中规则 2 的"许愿 vs 抱怨"边界、规则 4 的"褒义短评"例外、规则 5 的星级折算,
都是 v2 翻车后从人工标注中反推出来的——这正是校准实验的价值:
**prompt 的口径必须向标注者的口径对齐,而不是想当然**。

## 4. v2 为什么反而下降(实验中最有价值的发现)

v2 根据 v1 的 5 条错误加入了"点出具体问题→mixed""温吞表态→neutral"两条规则,
结果规则写得太粗,矫枉过正:

- "love the game… just add a season mode"(5星,人工=positive)被误判 mixed
  ——**许愿被当成了抱怨**
- "good"(3星,人工=positive)被误判 neutral——**明确褒义词被温吞规则吞掉**
- "it is okay"(5星,人工=positive)被误判 neutral——**信息量过少应按星级折算**

教训:错误驱动的 prompt 迭代**不保证单调提升**,每轮迭代必须回归全量校准集,
而不是只验证被修复的案例。

## 5. 方法论决策

- **few-shot 示例全部为错误模式的仿写**,不使用校准集原句——避免把"考题"
  塞进 prompt 造成评测泄漏、虚高准确率。
- **止步于 v3**:剩余 4 条分歧均为边界案例(极短评论"內容複雜"、褒贬权重
  判断等,人类标注者之间也可能存在分歧)。继续针对同一批 50 条调优,
  收益会来自对校准集的过拟合而非真实能力提升。

## 6. 剩余分歧(v3,4 条)

| 评论(节选) | 人工 | v3 | 性质 |
|------|------|-----|------|
| "內容複雜"(zh-TW,3星) | mixed | neutral | 极短评论,信息不足 |
| "nice but need steam account"(en,5星) | mixed | positive | 账号要求算"问题"还是"陈述"的边界 |
| "online gameplay is very bad, input lag…"(en,4星) | negative | mixed | 褒贬权重判断 |
| "是還好啦,畢竟免費也不會好到哪裡去…"(zh-TW,5星) | neutral | positive | 温吞表态 + 高星级的张力 |

## 7. 局限

- 样本量 n=50,±1 条 = ±2 个百分点,准确率数字应理解为区间而非精确值
- 校准仅覆盖 en/zh-TW;es/pt/fr 的表现未经人工验证
- 单标注者,无标注者间一致性(inter-annotator agreement)数据
- 更严谨的做法是保留一个未参与迭代的 held-out 集验证 v3,受样本量限制未做
